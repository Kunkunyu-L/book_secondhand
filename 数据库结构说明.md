# 二手书交易系统 - 数据库结构说明

## 数据库名称
`book_secondhand`

## 表结构总览

数据库共包含 **5张核心表**：
1. `user` - 用户表
2. `book_category` - 图书分类表
3. `platform_book` - 平台图书表
4. `user_book` - 用户图书表
5. `book_condition_price` - 图书条件和价格表（关联表）

---

## 1. user 表（用户表）

### 字段说明

| 字段名 | 类型 | 长度 | 是否为空 | 键 | 说明 |
|--------|------|------|----------|-----|------|
| id | bigint | 20 | NOT NULL | 🔑 主键 | 用户唯一ID（自增，无业务含义） |
| username | varchar | 30 | NOT NULL | | 登录账号（字母/数字/下划线，唯一） |
| password | varchar | 64 | NOT NULL | | 密码（加密存储，如BCrypt/SHA256） |
| phone | varchar | 11 | NULL | | 手机号（唯一，自动满足前送） |
| role | varchar | 20 | NULL | | 角色：user（普通用户）、customer_service（客服） |
| avatar | varchar | 255 | NULL | | 用户头像（移动端显示，存OSS/CDN地址） |
| nickname | varchar | 20 | NULL | | 用户的昵称 |
| credit_score | int | 11 | NULL | | 信用分（初始100，好评+1，差评-2，低于80限制发布） |
| status | tinyint | 1 | NULL | | 账号状态：1（正常）、0（禁用，违规/投诉过多） |
| account | int | 255 | NULL | | 用户余额，可用于提现 |
| created_at | datetime | 0 | NULL | | 注册时间 |
| updated_at | datetime | 0 | NULL | | 信息更新时间（管理更新等） |

### 业务说明
- 用户注册时必填：`username`、`password`、`phone`
- 密码使用 `bcryptjs` 加密存储
- 支持角色权限管理（普通用户/客服）
- 信用分机制：初始100分，好评+1，差评-2，低于80分限制发布商品
- 用户余额可用于提现

---

## 2. book_category 表（图书分类表）

### 字段说明

| 字段名 | 类型 | 长度 | 是否为空 | 键 | 说明 |
|--------|------|------|----------|-----|------|
| id | int | 11 | NOT NULL | 🔑 主键 | 分类唯一ID（自增主键） |
| img | varchar | 255 | NOT NULL | | 分类的图片 |
| name | varchar | 50 | NOT NULL | | 分类名称（如"计算机""文学小说"） |
| sort | int | 11 | NOT NULL | | 排序权重（数字越大，前端展示越靠前） |
| status | tinyint | 4 | NOT NULL | | 状态：1=启用（前端显示），0=禁用（仅后台可见） |
| created_at | timestamp | 0 | NOT NULL | | 创建时间 |
| updated_at | timestamp | 0 | NOT NULL | | 更新时间 |

### 业务说明
- 支持分类的启用/禁用控制
- 通过 `sort` 字段控制前端展示顺序
- 每个分类有对应的图片展示

### 字段说明

| 字段名 | 类型 | 长度 | 是否为空 | 键 | 说明 |
|--------|------|------|----------|-----|------|
| id | bigint | 20 | NOT NULL | 🔑 主键 | 平台图书ID（自增主键） |
| user_id | bigint | 20 | NULL | | 关联平台id |
| isbn | varchar | 13 | NOT NULL | | ISBN码（唯一标识，平台图书唯一） |
| title | varchar | 200 | NOT NULL | | 书名 |
| author | varchar | 100 | NOT NULL | | 作者 |
| publisher | varchar | 100 | NOT NULL | | 出版社 |
| publish_date | varchar | 200 | NULL | | 出版日期 |
| category | int | 11 | NULL | | 图书分类（关联 book_category.id） |
| tags | varchar | 100 | NULL | | 标签（逗号分隔，如"计算机,考研"） |
| status | varchar | 20 | NOT NULL | | 状态：onsale（在售）、offline（下架） |

- 状态管理：在售/下架

### 字段说明

| 字段名 | 类型 | 长度 | 是否为空 | 键 | 说明 |
|--------|------|------|----------|-----|------|
| id | bigint | 20 | NOT NULL | 🔑 主键 | 用户图书ID（自增主键） |
| user_id | bigint | 20 | NOT NULL | | 卖家用户ID（关联用户表user.id） |
| isbn | varchar | 13 | NOT NULL | | ISBN码（非唯一，同一ISBN可有多用户售卖） |
| title | varchar | 200 | NOT NULL | | 书名 |
| author | varchar | 100 | NOT NULL | | 作者 |
| publisher | varchar | 100 | NOT NULL | | 出版社 |
| publish_date | varchar | 200 | NOT NULL | | 出版日期 |
| category | int | 11 | NOT NULL | | 图书分类（关联 book_category.id） |
| tags | varchar | 100 | NULL | | 标签（逗号分隔） |
| status | varchar | 20 | NOT NULL | | 状态：onsale（在售）、sold（已售）、offline（下架） |
| cover_img | varchar | 255 | NULL | | 封面图URL（用户实拍图） |
| detail_imgs | text | 0 | NULL | | 详情图URL（逗号分隔，用户实拍） |
| book_story | text | 0 | NULL | | 卖家描述（如"考研核心资料,轻微笔记"） |

### 业务说明
- 用户发布的二手图书
- 同一ISBN可以有多个用户售卖（区别于平台图书）
- 支持用户实拍图片上传
- 有 `book_story` 字段，用户可以描述图书的个性化信息
- 通过 `nope`（付款次数）进行热门排序
- 状态：在售/已售/下架

---

## 5. book_condition_price 表（图书条件和价格表）

### 字段说明

| 字段名 | 类型 | 长度 | 是否为空 | 键 | 说明 |
|--------|------|------|----------|-----|------|
| id | bigint | 20 | NOT NULL | 🔑 主键 | 自增主键 |
| type | enum | 0 | NULL | | 关联类型：platform（平台）、user（用户） |
| book_id | bigint | 20 | NULL | | 关联主表ID（platform_book.id 或 user_book.id） |
| condition | tinyint | 4 | NOT NULL | | 成色（7-10分，7=较新，10=全新） |
| condition_desc | varchar | 500 | NULL | | 成色详述（如"首页有轻微折痕,内页完整"） |
| original_price | decimal | 10,2 | NOT NULL | | 原价（元，供参考） |
| price | decimal | 10,2 | NOT NULL | | 售价（平台统一价/用户自定义价） |
| stock | int | 11 | NOT NULL | | 库存（平台可批量，用户默认1） |

### 业务说明
- **核心关联表**：通过 `type` 和 `book_id` 关联 `platform_book` 或 `user_book`
- 一本书可以有多个成色和价格组合（通过此表实现）
- 成色评分：7-10分（7=较新，10=全新）
- 平台图书可以批量设置库存，用户图书默认库存为1
- 价格包含原价（参考价）和售价（实际售价）

```
  │

  │             └── 1:N ──> book_condition_price (图书价格和成色)
  │
  └── 1:N ──> user_book (用户图书)
                └── 1:N ──> book_condition_price (图书价格和成色)
```

## 核心业务逻辑

### 1. 图书展示逻辑
- **首页**：展示平台精选图书（`platform_book`）和用户售卖图书（`user_book`）
- **书市**：混合展示平台和用户图书，随机排序（`ORDER BY RAND()`）
- **分类**：通过 `book_category` 进行分类筛选

### 2. 价格和成色管理
- 通过 `book_condition_price` 表统一管理价格和成色
- 通过 `type` 字段区分是平台图书还是用户图书
- 支持一本书多个成色/价格组合

### 3. 用户权限
- 普通用户（`role='user'`）：可以发布和售卖图书
- 客服（`role='customer_service'`）：可能用于客服功能
- 信用分机制：低于80分限制发布

### 4. 状态管理
- **图书状态**：
  - `platform_book.status`: `onsale`（在售）、`offline`（下架）
  - `user_book.status`: `onsale`（在售）、`sold`（已售）、`offline`（下架）
- **分类状态**：`book_category.status`: 1（启用）、0（禁用）
- **用户状态**：`user.status`: 1（正常）、0（禁用）

---

## 数据库设计特点

1. **分离设计**：平台图书和用户图书分开存储，便于管理
2. **灵活定价**：通过 `book_condition_price` 表实现一本书多价格/成色
3. **信用体系**：用户信用分机制，保证交易质量
4. **状态管理**：完善的状态字段，支持商品生命周期管理
5. **扩展性**：预留了角色、余额等字段，便于后续功能扩展

---

## 注意事项

1. `book_condition_price` 表的 `type` 字段必须与对应的 `book_id` 匹配
2. 用户图书的 `nope`（付款次数）字段用于热门排序
3. 平台图书的 `sales_count`（销量）字段用于排序
4. 用户余额（`account`）字段已预留，但可能尚未实现相关业务逻辑
5. 图片字段存储URL，实际文件可能存储在OSS或CDN
