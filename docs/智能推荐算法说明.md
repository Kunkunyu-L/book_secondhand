# 智能推荐算法说明文档

## 一、算法概述

本项目实现了一个**基于多因子加权融合的混合推荐算法**，主要针对大学生二手书交易场景设计。

### 算法特点
- **轻量级**：无需复杂的机器学习框架，易于理解和维护
- **个性化**：基于用户专业进行精准推荐
- **实时性**：实时计算推荐分数，无需离线训练
- **可扩展**：模块化设计，便于添加新的推荐因子

---

## 二、算法架构

```
用户信息(User Profile)          图书信息(Item Profile)
      │                              │
      │  ┌──────────────────────────┐│
      └──►                          ││
         │  专业匹配得分 (40%)      ││
         │  分类权重得分 (20%)      ├─► 推荐分数计算
         │  热度得分 (25%)          ││     ↓
         │  时间衰减得分 (15%)      ││  排序输出
         │  成色加分 (10%)          ││     ↓
         └──►                       ││  Top-N推荐
            └──────────────────────────┘│
                     │                  │
                     └──────────────────┘
```

## 三、推荐分数计算公式

$$Score(w, u) = \sum_{i=1}^{5} W_i \times F_i$$

其中：
- $w$ = 图书（item）
- $u$ = 用户（user）
- $W_i$ = 第i个因子的权重
- $F_i$ = 第i个因子的得分

### 各因子详解

| 因子 | 权重 | 计算方式 | 说明 |
|------|------|----------|------|
| 专业匹配 | 40% | 关键词命中次数 × 40 | 书名/作者/标签中包含专业关键词 |
| 分类权重 | 20% | 分类系数 × 20 | 大学教材权重更高(1.2) |
| 热度得分 | 25% | min(销量×5, 25) | 销量/付款次数越高得分越高 |
| 时间衰减 | 15% | 30天内线性衰减 | 新发布图书得分更高 |
| 成色加分 | 10% | 成色值(7-10) | 成色越好得分越高 |

## 七、数据库依赖

算法依赖以下数据表：

1. **user表** - 用户专业信息（major字段）
2. **user_book表** - 用户发布的图书
3. **platform_book表** - 平台自营图书
4. **book_category表** - 图书分类
5. **book_condition_price表** - 图书成色价格信息

---

## 八、算法优势

1. **冷启动友好**：新用户只需填写专业即可获得推荐
2. **解释性强**：每个因子都有明确含义，便于调试优化
3. **性能优异**：单次查询毫秒级响应
4. **易于维护**：代码结构清晰，无需机器学习背景

---

## 九、可扩展方向

如需进一步提升推荐效果，可考虑：

1. **协同过滤**：基于用户-图书交互矩阵
2. **标签系统**：用户可自行添加兴趣标签
3. **行为追踪**：记录浏览、收藏、购买行为
4. **深度学习**：使用神经网络模型
