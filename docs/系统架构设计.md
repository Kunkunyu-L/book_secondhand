# 系统架构设计文档

## 一、系统概述

本系统是一个完整的二手书交易平台，采用 **前后端分离架构**，包含前台客户端（UniApp）、后台管理系统（Vue3）和后端服务（Node.js + Express）三大模块。

## 五、核心模块设计

### 5.1 认证授权模块

#### JWT 认证流程

```
┌─────────┐                 ┌─────────┐                 ┌─────────┐
│  客户端  │                 │  服务器  │                 │  数据库  │
└────┬────┘                 └────┬────┘                 └────┬────┘
     │                           │                           │
     │  1. POST /api/login       │                           │
     │  (username, password)     │                           │
     │─────────────────────────>│                           │
     │                           │  2. 验证用户信息           │
     │                           │─────────────────────────>│
     │                           │<─────────────────────────│
     │  3. 返回 JWT Token        │                           │
     │<─────────────────────────│                           │
     │                           │                           │
     │  4. 携带 Token 请求接口   │                           │
     │  Authorization: Bearer xxx│                           │
     │─────────────────────────>│                           │
     │                           │  5. 验证 Token             │
     │  6. 返回受保护资源        │                           │
     │<─────────────────────────│                           │
```

#### 权限控制（RBAC）

```
              └───────┬────────┘
                      │
              ┌───────▼────────┐
              │ chat_message   │
              │ chat_session   │
              │   (MySQL)      │
              └────────────────┘
```

#### 事件流

```javascript
// 客户端事件
socket.emit('create_session', { target_id, target_type })
socket.emit('send_message', { session_id, content, msg_type })
socket.emit('mark_read', { session_id })

// 服务器事件
io.on('connection', socket => {
  socket.on('send_message', handleSendMessage)
  socket.on('mark_read', handleMarkRead)
})

// 广播事件
io.to(sessionId).emit('new_message', messageData)
```

## 六、API 设计规范

### 6.1 RESTful API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /home/books/market | 获取图书列表 |
| POST | /home/books/detail | 获取图书详情 |

| PUT | /order/status | 更新订单状态 |
| DELETE /favorite/remove | 取消收藏 |

### 6.2 统一响应格式

```javascript
// 成功响应
{
  "status": 0,
  "message": "success",
  "data": { ... }
}

// 错误响应
{
  "status": 1,
  "message": "错误信息",
  "data": null
}
```

### 6.3 数据验证

使用 Joi 进行数据验证：

```javascript
const schema = Joi.object({
```

## 十、架构设计亮点

1. **前后端分离**：便于独立开发和维护
2. **跨平台前台**：UniApp 一次开发多端运行
3. **实时通信**：Socket.IO 支持即时聊天
4. **智能推荐**：多因子加权推荐算法
5. **RBAC 权限**：细粒度角色权限控制
6. **混合来源**：统一管理平台和用户图书
7. **订单快照**：保证历史数据完整性
