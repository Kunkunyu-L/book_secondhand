## 数据库初始化指南（从零重建）

> 适用于：本地开发环境需要**重新创建整个数据库**的情况。  
> 数据库名以下示例统一使用 `book_secondhand`，可按需修改。

---

### 一、准备工作

1. **先备份旧数据（可选但强烈推荐）**

```sql
-- 在 MySQL 客户端或工具中执行
-- 仅示例，请根据实际库名修改
mysqldump -u root -p book_secondhand > book_secondhand_backup.sql
```

2. 确认已经在根目录看到这几个 SQL 文件：

- `database_update.sql`
- `database_update_v2.sql`
- `database_update_v3.sql`

---

### 二、删除并重新创建数据库

在 MySQL 中执行：

```sql
-- 删除旧库（注意：会清空所有旧数据！）
DROP DATABASE IF EXISTS book_secondhand;

-- 新建库
CREATE DATABASE book_secondhand DEFAULT CHARSET utf8mb4;

USE book_secondhand;
```

---

### 三、按顺序执行 SQL 脚本

> 建议使用命令行或可视化工具（Navicat、DataGrip、MySQL Workbench 等）依次执行下面三个脚本。

#### 1. 执行基础脚本

```bash
mysql -u root -p book_secondhand < database_update.sql
```

该脚本主要包含：
- 用户、图书、订单等**核心业务表**；
- 最初版本的字段设计。

#### 2. 执行第二版脚本

```bash
mysql -u root -p book_secondhand < database_update_v2.sql
```

该脚本主要：
- 对部分表结构进行补充 / 调整；
- 增加与退款等相关的字段/表。

#### 3. 执行第三版脚本（当前最新）

```bash
mysql -u root -p book_secondhand < database_update_v3.sql
```

该脚本主要：
- 新增内容管理相关表（公告、帮助文档、轮播图等）；
- 新增客服相关表（聊天会话、消息、工单等）；
- 新增系统设置相关表（角色、配置、通知等）；
- 为 `user` 等表增加角色、客服相关字段。

> 如果执行过程中提示某些表已存在，一般是重复执行了脚本；全部**全新建库**的情况下，正常不会报重复表错误。

---

### 四、确认初始化结果

执行完三个脚本后，在 `book_secondhand` 库中应能看到（包括但不限于）：

- 核心：`user`、`platform_book`、`user_book`、`orders`、`order_items`、`book_condition_price`、`cart`、`address` 等
- 内容：`announcement`、`help_article`、`banner` 等
- 客服：`chat_session`、`chat_message`、`service_ticket` 等
- 系统：`admin_role`、`system_config`、`notification_template`、`notification` 等

如需查看详细字段，请直接打开对应 SQL 文件或 `数据库结构说明.md`。

---

### 五、与代码对应的说明

1. **后端服务（`server/`）**
   - 启动前请确认 `db/config.js` 中的库名、用户名、密码与本指南中创建的一致。
   - 重建数据库后，若没有任何管理员账户，需要根据脚本或手动插入一条管理员用户数据。

2. **前台 UniApp / 后台 Admin**
   - 所有接口默认指向 `book_secondhand` 库。
   - 重建数据库后，**历史测试数据会消失**，需重新注册账号、发布图书等。

---

### 六、可选：删除已废弃功能相关表

项目中「投诉举报」功能的前后台代码已经移除，但数据库中仍保留 `complaint` 表用于兼容旧数据。  
如果你是**全新部署**，且明确不再需要该表，可以在执行完所有脚本后手动删除：

```sql
USE book_secondhand;
DROP TABLE IF EXISTS complaint;
```

---

### 七、遇到问题时的排查方向

- **提示 “Unknown database 'book_secondhand'”**
  - 说明没有成功执行 `CREATE DATABASE`，请先创建库再导入。
- **提示 “Table 'xxx' doesn't exist”**
  - 说明某些脚本顺序错乱或执行中断，重新确认是否按本指南顺序执行了三个 SQL 文件。
- **后端启动报错 “ER_ACCESS_DENIED_ERROR” 或类似权限错误**
  - 检查 `db/config.js` 中的数据库账号密码是否正确、是否有访问 `book_secondhand` 的权限。
